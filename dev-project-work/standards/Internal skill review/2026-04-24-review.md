## 1. Topic-level gaps

Production concerns that don't have a file. Each entry notes whether any current file touches it superficially.

### Event-driven systems / messaging / async workflows
Message queues, pub/sub, event streams (Kafka, SQS, RabbitMQ, SNS), async job processing, dead-letter queues, message ordering / delivery guarantees (at-least-once vs exactly-once), consumer patterns, event sourcing, CQRS, outbox pattern, saga pattern, workflow orchestration (Temporal, Step Functions). Most non-trivial production systems use async processing for background jobs and cross-service integration; getting delivery guarantees and retry-ordering wrong produces data corruption.
*Existing coverage:* None. `resilience.md` touches idempotency keys; `error-handling.md` mentions async exception handling; nothing on architecture.

### Data backup, restore, and disaster recovery
Backup strategies (full / incremental / snapshot), point-in-time recovery, restore testing, RPO/RTO, multi-region DR, cross-region replication, geographic failover, data residency for DR, backup encryption, retention. "We have backups" is common; "we have tested backups" is rare, and the difference is catastrophic.
*Existing coverage:* None. `schema-evolution.md` is forward-only; `deployment.md` rolls back code, not data.
